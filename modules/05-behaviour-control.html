Behaviour control

Steering an agent with natural language: instructions, skills, and tools.

Reframe programming for this audience: you shape behaviour mostly by writing prose, not code. Markdown is the substrate. Filenames vary by vendor; the principle is general.

Prose is the program

Behaviour is specified in layered, plain-language instruction files.

Layers compose: hard rules, project context, task persona.

RulesNon-negotiable constraints: safety, style, source policy.
ContextProject vocabulary, audience, data model, local conventions.
PersonaThe working posture: concise, sceptical, teacherly, playful.

Vendor flavours: CLAUDE.md, AGENTS.md, context.md, "soul" files. Same idea, different filenames.

Stress principle over brand. This is reviewable, version-controllable and human-readable.

Why Markdown?

Readable

Humans and models can parse it.

Diffable

Behaviour changes show up in git.

Structured

Headings and lists give salient cues.

# Rules
- cite primary sources
- never hide uncertainty

## Context
- audience: STEM researchers
- examples should be local

## Persona
- concise, warm, sceptical
# behaviour is now reviewable
For researchers: this is specification as documentation. The artefact that controls the agent is the same artefact a colleague reads to understand it.

Skills - packaged competence

A skill is a reusable capability package. It tells the agent when to load extra instructions, scripts and resources.

Structure

name, description, trigger rules, instructions, optional scripts/resources.

Triggered by intent

The description is the match surface: "debug this" loads diagnosis; "grill me" loads a critique flow.

Examples

grill-with-docs, caveman, personas, skill-creator.

Skills = modularity for behaviour. Good place to mention this very project uses skills.

Tools - actions the model can take

A tool is procedural: the agent fills a typed input, external code runs, then the result returns to context.

{
  "name": "read_pdf",
  "description": "Extract text and tables",
  "input_schema": {
    "file": "path",
    "pages": "range"
  }
}
FormatsPDF, XLSX, DOCX, PPTX, CSV, images, audio.
ResultParsed content returns as structured context the model can reason over.

Skills shape how the agent behaves. Tools give it procedures and file-format access.

The schema is the contract. Forward-reference MCP as the standard way to expose tools across systems.
